n2c2

First place at the 2018 n2c2 Shared Task.

I led a winning team at the National NLP Clinical Challenges (n2c2, former i2b2).
We had to automatically classify clinical text of 202 patients into 13 selection criteria. We developed a rule-based classifier, a bag-of-words SVM classifier, and a bag-of-trichars Bi-LSTM neural network. We ranked first on the task (out of 46 participating teams) with a micro-averaged F1-score of 0.91.

  • Timespan: March–May 2018
  • Programming language: Java
  • https://github.com/bst-mug/n2c2

TREC-PM

First place in a TREC challenge.

The TREC Precision Medicine challenge was to rank documents from PubMed and ClinicalTrials.gov according to a set of 50 cancer patients. We employed weighted ElasticSearch dis_max queries to improve precision without a corresponding drop in recall. We had the best P@10 for biomedical articles and best infNDCG for clinical trials out of 27 participating teams.

  • Year: 2018
  • Programming language: Java
  • https://github.com/hpi-dhc/trec-pm

acres

Acronym resolution using word2vec.

We are currently exploring word2vec as a base model for acronym expansion, on top of which we are currently applying filtering rules, mainly to improve precision. Latent models such as word2vec improves upon distributional semantics approaches (such as n-gram matching) by making the model denser and therefore faster.

  • Timespan: 2018–2019
  • Programming language: Python
  • https://github.com/bst-mug/acres

ICDC

An automated ICD classifier.

Coding in the ICD (International Classification of Diseases) is error prone and time-consuming. We explored supervised approaches to automatically classify pathology reports from 19,000 cancer patients into 18 ICD code groups. We obtained the best results (micro-F1 = 0.82) with SVMs in a simple bag-of-words approach.

  • Timespan: 2013–2016
  • Programming language: Java
  • https://github.com/michelole/ICDClassifier

Recruit

A search engine for clinical trials.

I was involved with the design and development of an information retrieval system to find eligible patients for clinical trials. The system is in use since 2014 in a large cancer hospital in São Paulo area with data from more than 500,000 patients scattered over nine different databases. It uses ontologies to represent medical knowledge and integrate it with data from unstructured and structured sources.

  • Year: 2014
  • Programming language: PHP
